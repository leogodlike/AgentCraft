# 设计系统：华为风格

## 一、PPT 写作基础

### 写在前面：

- 观点在标题上，简短语句表达
- 每页核心观点 3-6 个，每个观点需有具体数据或案例支撑
- 字体：Noto Sans SC
- 字号三种以内
- 决不允许出现动画
- 符合公司视觉规范
- 关键内容标红加粗
- 慎用感叹号

### 内容充实度规范

#### 内容密度原则

- **观点提炼**：标题必须包含核心观点，避免空泛标题
- **完整解释**：每个观点需配有足够的正文内容支撑，禁止只有标题没有解释
- **信息密度**：单页内容应充实，避免过度留白，充分利用 1280×720px 的页面空间
- **逻辑闭环**：标题与正文需形成完整的论证链条，观点 → 论据 → 结论缺一不可

#### 内容组织方式

- **标题**：一句话概括核心观点（推荐类型：总结型/方案型/求助型/规划型）
- **正文**：3-5 个支撑要点，每个要点有具体说明和数据支撑
- **卡片充实**：每个卡片组件必须包含标题 + 关键数据/指标 + 简要说明，禁止只有标题没有内容的"空壳卡片"
- **数据支撑**：关键观点需有数据、案例或图表佐证，避免纯文字堆砌
- **视觉辅助**：使用架构图、流程图、数据图表增强信息传达效率

#### 反例与正例

| 类型    | 标题示例                                     | 正文内容                                                                                          | 评价                           |
| ------- | -------------------------------------------- | ------------------------------------------------------------------------------------------------- | ------------------------------ |
| ❌ 反例 | "项目进展"                                   | 仅列出 3 个任务名称                                                                               | 观点模糊，内容单薄，无数据支撑 |
| ❌ 反例 | "市场分析"                                   | 大段文字描述，无重点提炼                                                                          | 信息过载，观点不突出           |
| ✅ 正例 | "Q3 完成 3 大核心功能，用户活跃度提升 40%"   | 功能 1：XXX，DAU 提升 15%；功能 2：XXX，留存率提升 12%；下阶段计划：XXX                           | 观点明确，数据充实，逻辑完整   |
| ✅ 正例 | "传统产品 E2E 流程耗时过长，需优化 3 个断点" | 痛点：平均耗时 72 小时；根因：系统 A/B/C 数据不互通；方案：建设统一数据中台；预期：缩短至 24 小时 | 问题-根因-方案-预期完整闭环    |

#### 单页内容量建议

- **最少内容**：至少包含 1 个核心观点 + 4 个支撑要点 + 1 个数据/图表，每个要点需有具体说明（非纯标题）
- **适中内容**：1 个核心观点 + 4-5 个支撑要点 + 2-3 个数据/图表
- **最大内容**：不超过 6 个支撑要点，避免信息过载，复杂内容拆分为多页

### 如何搭好 PPT 骨架 – 逻辑

1. 如何构建逻辑——遵循金字塔原理

   - 确认目标 → 各角度分解 → 如何执行、落地
   - 现状问题 → 方案框架 → 方案展开

2. 逻辑套路——结构化呈现 Tips：

   - 问题解决: 痛点 → 根因 → 改进方案 → 落地计划 → 决策/求助
   - 问题分析: 概况 → 分析 → 下一步计划
   - 规划: 背景 → 价值 → 工作沙盘 → 落地计划 → 决策/自助
   - 方案: 现状问题 → 方案框架 → 方案展开
   - 个人总结: 自我介绍 → 工作总结 → 下一步计划 → 思考与建议

### 如何组织 PPT 内在 – 内容

- **一句话总结本页内容**：

  - 类型分类：

    - **总结型**: 适用于年度/半年度总结（多讲数据），示例：突破三大挑战，让更多业务实现"东数西算"
    - **求助性**：适用于推动问题改进，示例：传统产品涉及平台多、断点多，E2E 流程耗时长
    - **方案型**：适用于适用于立项汇报，示例：大云能力强化服务基座，结合企业差异化特性，构筑最强混合云基础服务

  - 有逻辑：向上服务于故事主线，标题串起来是完整故事；向下服务于正文内容，符合标题/正文一致性原则

  - 有观点：听众可以通过标题阅读跳过内容。论证类标题：高度总结事实；论述类标题：高度总结洞见及下一步计划

- **怎么写正文**

  1. 定框架：确定页面布局（推荐使用占位法）
  2. 填内容：先填空，后精简
  3. 再加工：突出重点，优化样式（推荐使用占位法）

- **经典结构**：上方：标题，左侧：能力沙盘，右上：典型项目，右下：服务能力规划

## 二、排版规范与布局

### 文本与段落样式

**字体大小定义（Tailwind CSS 类）**：

直接使用 Tailwind 任意值类名，无需扩展配置：

- **页面标题**：`class="text-[35px]"`（35px）
- **一级标题/卡片标题**：`class="text-[23px]"`（23px）
- **二级标题/内容文本**：`class="text-[21px]"`（21px）
- **正文文本**：`class="text-[19px]"`（19px）
- **辅助文本（图注、角标等）**：`class="text-[16px]"`（16px）

- **行距规范**：

  - 默认段落行距 **1.5 倍**
  - 特殊情况可调整至 1.6 倍和 2.0 倍，两端对齐
  - 端前间距一般为 0，端后间距一般为 12 像素，可根据实际内容量调整端后间距，常用 3、6、12、18 像素

- **文本颜色**：

  - 常规文本：通常使用黑色
  - 强调文字使用红色，在文本中使用红色强调内容时，一般不再加粗该内容
  - 极少数非重点内容和注释使用灰色
  - **禁止使用色彩太淡的灰色文字**：为保证可读性，文字颜色应保证足够的对比度。禁止使用 `#a6a6a6` (华为灰 2) 及更淡的灰色作为正文或主要注释文字颜色。如需使用灰色文字，建议使用 `#8c8c8c` (华为灰 1) 且仅用于页码、英文副标题等辅助信息

- **标题样式**：

  - 通常使用黑色，在红头卡片内通常为反白色
  - 主要标题通常加粗处理，小标题不建议加粗处理

- **段落样式**：
  - 使用项目符号和编号一般使用默认样式
  - 结合卡片使用时，需调整边距，不使用项目符号和编号时，左右边距保持一致常用 12、18、24 像素
  - 使用项目符号和编号时，通常左边距少于右边距 6 像素
  - 灵活调整上边距从而调整内容在卡片中的位置
  - 结合文本规范，内容较多时，大段文字不建议使用 19px 或更大的字号

## 三、标准页面结构

### 基础布局框架

```
┌─────────────────────────────────────────────────────────┐
│ [顶部装饰线] 1px 红色线条                                │
├─────────────────────────────────────────────────────────┤
│ [标题栏] 高度 90px                                       │
│  ├─ 红色指示条 (4px × 30px)                             │
│  ├─ 主标题 (35px 加粗)                                  │
│  ├─ 英文副标题 (19px 灰色)                              │
│  └─ 页码 (右对齐)                                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ [内容区域] 高度约 504px，左右边距一致                     │
│  ├─ 左侧导航栏 (可选，固定宽度 130px)                    │
│  └─ 主内容区 (flex 自适应)                               │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 页面类型与结构

#### 1. 架构图页面

- **用途**：展示系统分层架构、组织架构
- **结构**：左侧垂直导航 + 右侧分层网格布局
- **特点**：层级清晰，模块边界明确，使用红色导航条标识层级名称
- **示例**：触点层 → 应用层 → 双中台 → 云平台

#### 2. 数据对比页面

- **用途**：月度/季度数据对比、效果分析
- **结构**：顶部数据表格 + 底部图表组合
- **特点**：数据表格与可视化结合，对比维度清晰（如 5A 模型）
- **图表类型**：柱状图、蝴蝶图、折线图

#### 3. 时间线页面

- **用途**：活动策划、项目进度、里程碑展示
- **结构**：横向时间轴 + 活动内容网格
- **特点**：时间顺序清晰，活动内容一目了然
- **元素**：时间节点标签 + 内容卡片 + 状态标识

#### 4. 品牌策略页面

- **用途**：品牌定位、市场策略、核心能力展示
- **结构**：中央标题 + 卡片式布局
- **特点**：重点突出，层次分明，视觉聚焦
- **元素**：核心标题 + 能力卡片 + 价值主张

#### 5. 运营体系页面

- **用途**：用户运营、会员体系、服务流程
- **结构**：流程式布局 + 指标说明
- **特点**：逻辑清晰，可操作性强，步骤明确
- **元素**：流程节点 + 关键指标 + 说明文字

## 四、组件样式库

### 1. 顶部装饰线

- **样式**：1px 实线，红色 (`#c7020e`)
- **用途**：页面顶部视觉分隔
- **技术实现**：`<div class="w-full h-[1px] bg-huaweiRed"></div>`

### 2. 标题栏

- **样式**：高度 90px，白色背景，底部灰色边框
- **元素**：
  - 红色指示条：4px × 30px
  - 主标题：35px，黑色，加粗
  - 英文副标题：19px，灰色
  - 页码：19px，灰色，右对齐
- **技术实现**：

```html
<div
  class="w-full h-[90px] bg-white border-b border-huaweiGray4 flex items-center px-8 gap-3"
>
  <div class="w-[4px] h-[30px] bg-huaweiRed"></div>
  <span class="text-[35px] font-bold text-black">页面标题</span>
  <span class="text-[19px] text-huaweiGray1 ml-1">Page Title</span>
  <span class="ml-auto text-[19px] text-huaweiGray3">01</span>
</div>
```

### 3. 红色导航条

- **样式**：红色背景 (`#c7020e`)，白色文字，固定宽度 130px
- **用途**：页面左侧或顶部导航，标识层级名称
- **元素**：
  - 中文标题：19px，白色，加粗
  - 英文副标题：15px，白色/70%透明度
- **技术实现**：

```html
<div
  class="w-[130px] bg-huaweiRed flex flex-col justify-center pl-3 flex-shrink-0"
>
  <span class="text-[23px] text-white font-bold">触点</span>
  <span class="text-[19px] text-white/70 mt-0.5">Touchpoints</span>
</div>
```

### 4. 数据卡片（带左侧红条）

- **样式**：白色背景，灰色边框 (`#d9d9d9`)，左侧 4px 红色边框
- **用途**：展示关键数据、功能模块、应用系统
- **元素**：
  - 主标题：17px，黑色，加粗
  - 英文副标题：12px，灰色
  - 关键指标：19px，红色，加粗（展示核心数据）
  - 增长/对比：12px，灰色（辅助数据说明）
  - 简要说明：12px，黑色/80%透明度（一行概括功能或成果）
- **技术实现**：

```html
<div
  class="bg-white border border-huaweiGray4 border-l-[4px] border-l-huaweiRed
            flex flex-col justify-start pl-3 py-2.5"
>
  <span class="text-[21px] font-bold">营销管理系统</span>
  <span class="text-[16px] text-huaweiGray1 mt-0.5">Marketing Mgmt</span>
  <div class="mt-2 flex items-baseline gap-2">
    <span class="text-[23px] font-bold text-huaweiRed">DAU 12.8万</span>
    <span class="text-[16px] text-huaweiGray1">↑15% 同比增长</span>
  </div>
  <p class="text-[16px] mt-1.5 text-black/80">覆盖全渠道营销自动化，支持 200+ 营销场景</p>
</div>
```

### 5. 灰色标签

- **样式**：灰色边框 (`#bfbfbf`)，浅灰背景 (`#f2f2f2`)，黑色文字
- **用途**：分类标签、状态标识
- **技术实现**：`<span class="border border-huaweiGray3 bg-huaweiGray5 px-2 py-1 text-[19px]">标签</span>`

### 6. 红色标签

- **样式**：红色边框 (`#c7020e`)，浅红背景 (`#fff1ef`)，红色文字
- **用途**：强调标签、关键指标、重要状态
- **技术实现**：`<span class="border border-huaweiRed bg-huaweiRedBg text-huaweiRed px-2 py-1 text-[19px]">标签</span>`

### 7. 数据表格

- **样式**：白色背景，灰色边框，表头红色背景
- **用途**：数据对比、指标展示
- **元素**：
  - 表头：红色背景，白色文字，15px（`.text-[19px]`）
  - 数据行：白色背景，黑色/红色文字，17px（`.text-[21px]`）
  - 强调数据：红色加粗
- **技术实现**：使用 flex 或 table 布局，边框统一使用 `#d9d9d9`

### 8. 图表容器

- **样式**：白色背景，灰色边框，标题居中或左对齐
- **用途**：数据可视化展示
- **集成**：使用 ECharts 实现柱状图、折线图、饼图等
- **图表颜色规范（强制）**：
  - 坐标轴标签 (`axisLabel.color`)：使用 `#2B2D42` (深色) 或 `#000000` (黑色)
  - 图例文字 (`textStyle.color`)：使用 `#2B2D42` 或 `#000000`
  - 数据标签 (`label.color`)：使用 `#2B2D42`、`#000000` 或 `#c7020e` (华为红)
  - 分割线 (`splitLine.lineStyle.color`)：使用 `#d9d9d9` (华为灰 4)
  - **禁止**在图表中使用 `#a6a6a6` (华为灰 2) 或更淡的灰色作为文字颜色
- **技术实现**：

```html
<div class="border border-huaweiGray4 bg-white p-3 flex flex-col overflow-hidden">
  <div id="chart-id" class="flex-1 min-h-0 w-full"></div>
</div>
```

**ECharts 颜色配置示例**：

```javascript
chart.setOption({
  xAxis: {
    axisLabel: { fontSize: 12, color: '#2B2D42' },  // ✅ 深色文字
    // axisLabel: { color: '#a6a6a6' },  // ❌ 禁止浅灰色
  },
  yAxis: {
    axisLabel: { fontSize: 12, color: '#2B2D42' },
    splitLine: { lineStyle: { color: '#d9d9d9' } },  // ✅ 分割线可用灰色
  },
  series: [{
    label: { color: '#c7020e' },  // ✅ 数据标签用华为红
  }]
});
```

### 9. 时间线节点

- **样式**：红色标题 + 灰色背景内容区
- **用途**：时间轴上的活动/任务展示
- **元素**：
  - 时间节点标签：红色背景或红色文字
  - 内容区：灰色背景 (`#f2f2f2`)，白色卡片
- **技术实现**：使用 grid 布局实现横向时间轴

### 10. 箭头与连接线

- **样式**：1px 宽度线条，箭头大小选最小
- **用途**：流程指示、层级关系
- **颜色**：根据场景使用红色（强调）或灰色（普通）
- **技术实现**：使用 CSS 或 SVG 实现

## 五、常用组件样式汇总

- 卡片形状轮廓多以浅灰框、深灰框、红色框为主；形状填充多以纯白、浅灰、红色为主。线框粗细多使用 1px，除特殊需要，尽量不使用其他粗细。所有卡片、按钮、容器等元素的边角禁止使用圆角，必须使用直角（图表中的圆形元素如饼图、环形图等不受此限制）。

- 箭头线、连接线、分割线等线条类元素，为保证统一性，尽量使用 1px 宽度的线条，箭头连接类型尽量参考右侧样式，箭头大小一般选最小。

- 其他线性元素，多使用 1px 宽度的线条呈现，描边使用不同程度的灰色、黑色、或红色，填充一般使用不同程度的灰色、或渐变灰色和红色，具体使用方式可参考右侧样例。

## 六、建议配色

### 配色原则

- 为简化制作 PPT 难度，增强统一性，建议减少配色复杂度，常规情况下使用红色系及无色系配色，特殊情况下可使用异色系配色
- 如非必要，一般线框、线条、图形等元素使用灰色系配色；
- 文字建议使用黑色，非特殊情况不使用灰色；
- **禁止使用太淡的灰色文字**：正文和主要注释必须使用黑色或深色文字，确保可读性。`#a6a6a6` (华为灰 2) 及以上淡色只能用于非必要的装饰性元素；如需使用灰色文字，建议使用 `#8c8c8c` (华为灰 1) 且仅用于页码、英文副标题等辅助信息
- 强调类文字、标题、强调类线框可酌情使用红色；部分架构图，为区分内容可使用红色系，配色强调主要部分。不建议大量使用红色，易造成页面混乱
- **禁止使用渐变和阴影**：所有页面（除封面页、目录页、过渡页、尾页外）都禁止使用渐变和阴影效果
- **图表文字禁止浅灰色**：ECharts 图表中的坐标轴标签、图例文字、数据标签等所有文字元素必须使用黑色 (`#2B2D42` 或 `#000000`) 或华为红 (`#c7020e`)，禁止使用浅灰色（`#a6a6a6` 及更淡）作为图表文字颜色。图表网格线/分割线可使用 `#d9d9d9` (华为灰 4)

## 八、建议布局参考线

- 常规布局情况，建议内容按照本页参考线分栏，具体布局样式，可参考本文档"常见布局参考样式"页。
